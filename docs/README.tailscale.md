## Вариант 1: Tailscale (рекомендую) ⭐

Простая VPN-сетка. Бесплатно, без открытия портов, шифрование из коробки.

---

### На компе

1. **Скачай** Tailscale с [официального сайта](https://tailscale.com/download) (или через `winget install Tailscale.Tailscale`)
2. **Установи** — запусти MSI-файл, установка стандартная
3. **Запусти** — появится иконка в системном трее (рядом с часами)
4. **Войди** в аккаунт (Google / GitHub / Microsoft — любой). Откроется браузер, подтверди вход
5. **Узнай свой Tailscale-IP** — кликни по иконке в трее → увидишь адрес вида `100.x.x.x`

---

### На телефоне

6. **Установи Tailscale:**
   - **Google Play** / **App Store** — если магазин приложений доступен
   - Если нет — используй APK из репозитория: скопируй файл `com.tailscale.ipn_564.apk` на телефон и открой его для установки
7. **Войди в тот же аккаунт**, что на компьютере
8. **Включи Tailscale** — переключатель должен стать зелёным. Появится иконка VPN — это нормально

---

### В прокси

9. **Открой настройки tg-ws-proxy** (правый клик по иконке в трее → Настройки, или через аргументы командной строки)
10. В поле `host` укажи `0.0.0.0` (вместо `127.0.0.1`)
11. **Проверь брандмауэр Windows:**
    - Пуск → `Брандмауэр Защитника Windows` → Дополнительные параметры
    - Создать правило для входящего подключения → порт TCP `1443` → разрешить

---

### В Telegram на телефоне

12. Открой Telegram → **Настройки** → **Продвинутые** → **Тип подключения** → **Прокси**
13. Нажми **Добавить прокси**
    - **Тип:** `MTProto`
    - **Сервер:** Tailscale-адрес компьютера (например `100.113.233.74`)
    - **Порт:** `1443`
    - **Secret:** тот же, что в логах tg-ws-proxy (32 hex-символа)
14. Включи прокси

---

### Важно

- Tailscale сам пробьёт NAT, порты на роутере открывать **не нужно**
- Иконка VPN на телефоне — это нормально, WireGuard так работает
- Весь остальной трафик (сайты, YouTube) идёт напрямую, а не через Tailscale
- Если Tailscale выключен на телефоне — прокси работать не будет
- **Компьютер должен быть включён.** Если комп выключен или в спящем режиме — прокси не работает, так как он запущен на нём
